# Feature: RAG 知识库前端界面

## Problem Statement

RAG 知识库核心模块已完成（spec-rag-core），提供了一组 API 用于文档上传、切片预览、确认入库和语义检索。但当前这些能力只能通过 curl 或 Swagger 调用，普通用户无法使用。需要一个前端交互界面，让教师和学生能直观地上传文档、管理知识库内容，并对库中内容进行检索。

## Success Metrics

- 从选择文件到看到切片预览，整个交互不超过 5 步（选文件 → 点上传 → 看预览 → 调参数 → 确认入库）
- 文件上传成功率和入库成功率在前端有明确的状态反馈，失败时有中文错误提示
- 搜索结果以列表形式清晰展示，每条结果包含内容摘要、匹配度和来源信息
- 页面加载和状态切换不白屏、不卡顿，按钮点击后立即显示 loading 状态

## User Stories

- 作为教师，我登录系统后可以上传一本名著的 TXT 文件，预览它的自动切片效果，再确认导入知识库
- 作为教师，我可以在知识库中搜索一个话题，看到返回的相关原文片段和它们的匹配度
- 作为学生，我可以搜索知识库中的内容，查看原文片段和它来自哪本书
- 作为管理员，我可以看到知识库中已导入的文档列表，知道库里有内容还是空的

## Acceptance Criteria

- [ ] 侧边栏有"知识库"导航项，所有登录角色（学生/教师/管理员）可见
- [ ] 在知识库页面可以选择一个 .txt 文件，上传后预览前 5 个切片的内容和序号
- [ ] 预览后可以调整切片参数（大小/重叠量/向量模型），点击确认将文档导入知识库
- [ ] 入库前有确认环节，入库成功后有明确的成功提示（包含切片数量和所用模型）
- [ ] 搜索输入框支持回车提交和按钮点击两种触发方式
- [ ] 搜索结果按匹配度降序排列，每条结果展示：内容摘要、匹配度（百分比/进度条）、来源文件名、第几片/共几片
- [ ] 知识库为空时搜索给出友好提示，而非空白或报错
- [ ] 上传区域支持点击选择和拖放两种方式选文件，仅接受 .txt 文件

## Non-Goals

- 本期不做文档管理功能（查看已导入文档列表、删除文档、更新文档）
- 本期不做多文档批量上传（一次只传一个文件）
- 本期不做 PDF/Word 等非 TXT 格式支持（上传时仅接受 .txt）
- 本期不做搜索结果的翻页或无限滚动（最多显示 Top 5）
- 本期不做搜索结果展开/折叠全文交互（仅展示内容摘要）
- 本期不做角色权限区分（所有登录角色使用相同的上传和搜索功能）

## Constraints

- 所有 API 调用需要携带 JWT Token（使用当前登录状态的认证头）
- 上传入口仅接受 .txt 格式文件，大小不超过 10MB
- 文件不保留在服务器端（入库即清理），预览后未确认入库的文件不可恢复
- 前端不依赖任何新的第三方库或 CDN 资源
