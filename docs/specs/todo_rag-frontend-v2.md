# Feature: RAG 知识库增强 — 入库反馈 + 文档管理（前端）

## Problem Statement

当前 RAG 知识库前端已完成基本的"上传 → 预览 → 确认入库 → 搜索"流程，但在实际使用中存在两个明显不足：

1. **入库过程缺乏反馈**：点击"确认入库"后，按钮显示 spinner 和"入库中..."文字，但后端实际在同步执行切片、生成 Embedding（DashScope API 调用）、写入 Chroma、重建 BM25 索引，整个过程可能耗时数秒到数十秒。用户看不到当前进度（切片中？向量生成中？索引重建中？），完成后也没有明确的成功提示（仅内联绿色卡片，无弹窗），不清楚是否真的完成。

2. **无法查看库内内容**：知识库当前只有"入库"和"搜索"两个入口。用户入库后无法看到库中已有哪些文档、每个文档多少片、用什么模型入库的。想删除不需要的文档也没有入口。这个模块叫"知识库"，但只能往里塞东西，不能查看和管理。

## Success Metrics

- 点击确认入库后，用户能在按钮/loading 区看到阶段性的描述文字（如"正在切片…"、"正在生成向量…"、"正在重建索引…"），而非仅一个通用 spinner
- 入库成功后，既有内联绿色卡片，也有 ElMessage.success 弹出提示
- 入库失败后，ElMessage.error 显示具体原因，状态重置到可重试状态
- 进入知识库页面后，自动加载并显示库中已有文档列表
- 每个文档条目展示文件名、切片数、向量模型、入库时间
- 删除文档有二次确认弹窗，删除后列表自动刷新
- 文档列表为空时显示明确的空状态提示，引导用户上传

## User Stories

- 作为教师，我上传文件并点击确认入库后，能清晰看到当前处理到哪一步了（切片/向量/索引），完成后明确知道是否成功
- 作为教师，我想看看知识库里现在有哪些文档，每个文档多大量，是什么时候入库的
- 作为教师，我可以删除不需要的文档，清理知识库空间
- 作为管理员，我可以在知识库首页看到库的概览状态（文档总数、切片总数）

## Acceptance Criteria

- [ ] 确认入库按钮点击后，ragLoading 为 true，按钮文字分阶段变化（"正在切片…" → "正在生成向量…" → "正在重建索引…"）
- [ ] 入库成功后，触发 ElMessage.success('入库成功')，同时显示内联绿色卡片
- [ ] 入库失败后，ElMessage.error 显示后端返回的错误详情，状态可重试
- [ ] 进入知识库页签时，自动调用 list API 加载文档列表
- [ ] 文档列表以表格形式展示：文件名、切片数、模型、入库时间
- [ ] 每个文档有"删除"操作，点击后弹 ElMessageBox.confirm 确认
- [ ] 确认删除后调用后端 delete API，成功后列表刷新 + ElMessage.success
- [ ] 删除失败时 ElMessage.error 显示错误原因
- [ ] 文档列表为空时，显示空状态引导"知识库为空，请上传文档"
- [ ] 搜索面板在有文档和无文档时分别给出恰当的提示

## Non-Goals

- 本期不做流式/SSE 入库进度推送（后端保持同步 API）
- 本期不做多文件批量上传
- 本期不做 PDF/Word 等非 TXT 格式支持
- 本期不做文档编辑（重命名、重新上传覆盖）
- 本期不做版本管理或历史记录

## Constraints

- 前端不引入新的第三方 CDN 依赖
- 确认入库的"分阶段提示"在前端用 setTimeout 模拟（真实进度需要后端 SSE，当前不实现）
- JWT 认证要求不变
