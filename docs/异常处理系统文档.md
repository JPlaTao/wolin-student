# 全局异常处理系统文档

## 概述

本项目实现了一个完整的全局异常处理系统，为FastAPI应用提供统一的错误处理机制，提高系统的健壮性和可维护性。

## 系统架构

### 1. 自定义异常类 (`core/exceptions.py`)

定义了一系列自定义异常类，继承自基础异常类 `AppException`：

#### 基础异常类
- **AppException**: 所有自定义异常的基类
  - 提供统一的异常属性：code, message, detail, status_code, extra
  - 支持 `to_dict()` 方法转换为字典格式

#### 业务异常类
- **BusinessException**: 业务逻辑异常 (HTTP 400)
- **ValidationException**: 参数验证异常 (HTTP 422)
- **NotFoundException**: 资源未找到异常 (HTTP 404)
- **ConflictException**: 冲突异常 (HTTP 409)

#### 认证授权异常类
- **UnauthorizedException**: 未认证异常 (HTTP 401)
- **ForbiddenException**: 权限不足异常 (HTTP 403)
- **TokenExpiredException**: Token过期异常 (HTTP 401)
- **TokenInvalidException**: Token无效异常 (HTTP 401)

#### 数据库异常类
- **DatabaseException**: 数据库操作异常 (HTTP 500)
- **DuplicateKeyException**: 重复键异常 (HTTP 409)

#### 外部服务异常类
- **ExternalServiceException**: 外部服务异常 (HTTP 502)

### 2. 错误码系统

使用枚举 `ErrorCode` 定义统一的错误码：

```python
class ErrorCode(str, Enum):
    # 通用错误码 (1000-1999)
    SUCCESS = "0000"
    INTERNAL_ERROR = "1000"
    INVALID_REQUEST = "1001"
    
    # 业务错误码 (2000-2999)
    BUSINESS_ERROR = "2000"
    VALIDATION_ERROR = "2001"
    NOT_FOUND = "2002"
    CONFLICT = "2003"
    
    # 认证授权错误码 (3000-3999)
    UNAUTHORIZED = "3000"
    FORBIDDEN = "3001"
    TOKEN_EXPIRED = "3002"
    
    # 数据库错误码 (4000-4999)
    DATABASE_ERROR = "4000"
    DUPLICATE_KEY = "4001"
```

### 3. 全局异常处理器 (`core/exception_handlers.py`)

实现了多种异常处理器：

#### 自定义异常处理器
- `app_exception_handler`: 处理所有 `AppException` 子类

#### 参数验证异常处理器
- `validation_exception_handler`: 处理FastAPI请求参数验证异常
- `pydantic_validation_exception_handler`: 处理Pydantic模型验证异常

#### 数据库异常处理器
- `sqlalchemy_exception_handler`: 处理SQLAlchemy数据库异常
  - 自动识别重复键错误 (返回 HTTP 409)
  - 自动识别外键约束错误 (返回 HTTP 400)

#### 认证异常处理器
- `jwt_exception_handler`: 处理JWT相关异常

#### HTTP异常处理器
- `http_exception_handler`: 处理FastAPI的HTTPException

#### 通用异常处理器
- `general_exception_handler`: 处理所有未捕获的异常
  - 开发环境显示详细错误
  - 生产环境隐藏敏感信息

### 4. 标准错误响应格式

所有异常都返回统一的JSON响应格式：

```json
{
  "code": "2001",
  "message": "参数验证失败",
  "detail": "用户名不能为空",
  "timestamp": "2024-04-16T12:00:00",
  "request_id": "abc12345",
  "extra": {
    "field": "username"
  }
}
```

### 5. 敏感信息过滤

系统自动过滤敏感信息，包括：
- 数据库连接字符串 (mysql://, postgresql:// 等)
- 密码相关字段 (password, secret)
- Token和API密钥 (token, api_key)

## 使用方法

### 1. 抛出自定义异常

```python
from core.exceptions import (
    ValidationException,
    NotFoundException,
    BusinessException,
    UnauthorizedException
)

# 参数验证异常
raise ValidationException(
    message="参数验证失败",
    field="username",
    detail="用户名不能为空"
)

# 资源未找到异常
raise NotFoundException(
    resource="用户",
    detail=f"ID为 {user_id} 的用户不存在"
)

# 业务异常
raise BusinessException(
    message="业务操作失败",
    detail="当前状态不允许执行此操作"
)

# 认证异常
raise UnauthorizedException(
    message="未认证",
    detail="请先登录"
)
```

### 2. 在API中使用

```python
from fastapi import APIRouter, Depends
from core.exceptions import NotFoundException, BusinessException
from utils.log_decorators import log_api_call

@router.post("/register")
@log_api_call("用户注册")
def register(user: UserCreate, db: Session = Depends(get_db)):
    # 业务校验
    if user.role not in valid_roles:
        raise ValidationException(
            message=f"角色必须是以下之一: {', '.join(valid_roles)}",
            field="role"
        )
    
    # 检查重复
    if db.query(User).filter(User.username == user.username).first():
        raise ConflictException(
            message="用户名已存在",
            detail="该用户名已被其他用户使用"
        )
    
    # 执行业务逻辑
    new_user = User(username=user.username, ...)
    db.add(new_user)
    db.commit()
    return {"msg": "注册成功"}
```

### 3. 在Service层使用

```python
from core.exceptions import ValidationException

def validate_counselor(counselor_id: int, db: Session):
    counselor = db.query(Teacher).filter(
        Teacher.teacher_id == counselor_id,
        Teacher.role == 'counselor'
    ).first()
    
    if not counselor:
        raise ValidationException(
            message=f"教师 ID {counselor_id} 不存在或不是 counselor 角色",
            field="advisor_id",
            detail="请选择有效的辅导员老师"
        )
```

### 4. 在Auth层使用

```python
from core.exceptions import UnauthorizedException, ForbiddenException

async def get_current_admin_user(current_user: User = Depends(get_current_user)):
    if current_user.role != "admin":
        raise ForbiddenException(
            message="权限不足",
            detail="仅管理员可访问此资源"
        )
    return current_user
```

## 系统特点

### 1. 统一的错误响应格式
- 所有异常都返回相同结构的JSON响应
- 包含错误码、消息、详情、时间戳、请求ID等

### 2. 自动化的异常处理
- 无需在每个函数中手动处理异常
- 全局异常处理器自动捕获和处理所有异常

### 3. 敏感信息保护
- 自动过滤数据库连接字符串、密码、Token等敏感信息
- 生产环境隐藏详细错误信息

### 4. 智能的错误识别
- 自动识别数据库重复键错误
- 自动识别外键约束错误
- 根据异常类型自动返回合适的HTTP状态码

### 5. 完善的日志记录
- 预期内的异常记录为warning
- 未预期的异常记录为error（包含堆栈信息）
- 包含请求ID，便于问题追踪

### 6. 灵活的扩展性
- 可以轻松添加新的自定义异常类
- 可以为特定异常类型定制处理器

## 最佳实践

### 1. 选择合适的异常类型
- 使用 `ValidationException` 处理参数验证错误
- 使用 `NotFoundException` 处理资源不存在的情况
- 使用 `BusinessException` 处理业务逻辑错误
- 使用 `UnauthorizedException` 处理认证失败
- 使用 `ForbiddenException` 处理权限不足

### 2. 提供详细的错误信息
```python
# ❌ 不好的做法
raise NotFoundException("用户不存在")

# ✅ 好的做法
raise NotFoundException(
    resource="用户",
    detail=f"ID为 {user_id} 的用户不存在"
)
```

### 3. 避免过度使用异常
```python
# ❌ 不好的做法
def check_user_exists(user_id):
    if not db.query(User).filter(User.id == user_id).first():
        raise NotFoundException(resource="用户")

# ✅ 好的做法
def get_user(user_id):
    user = db.query(User).filter(User.id == user_id).first()
    if not user:
        raise NotFoundException(
            resource="用户",
            detail=f"ID为 {user_id} 的用户不存在"
        )
    return user
```

### 4. 在适当层级处理异常
- **Controller层**: 处理HTTP相关的异常
- **Service层**: 处理业务逻辑异常
- **DAO层**: 处理数据库异常（通常由全局处理器处理）

## 总结

这个全局异常处理系统为项目提供了：

1. **统一的错误处理机制**：所有异常都通过统一的流程处理
2. **标准化的错误响应**：客户端收到格式一致的错误信息
3. **完善的错误分类**：根据异常类型自动返回合适的HTTP状态码
4. **智能的错误识别**：自动识别和处理常见错误模式
5. **安全的错误信息**：保护敏感信息不被泄露
6. **详细的日志记录**：便于问题追踪和调试

通过这个系统，开发者可以专注于业务逻辑的实现，而无需担心异常处理的细节，大大提高了开发效率和代码质量。
