# 配置说明文档

本文档解释 `config.json` 中所有配置项的含义和使用方式。

## 快速开始

1. 复制 `config.example.json` 为 `config.json`
2. 根据实际情况修改配置项
3. 运行项目

---

## 配置项详解

### 1. database - 数据库配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `driver` | string | `mysql+pymysql` | 数据库驱动，MySQL 使用 `mysql+pymysql` |
| `host` | string | `localhost` | 数据库服务器地址 |
| `port` | int | `3306` | 数据库端口，MySQL 默认 3306 |
| `username` | string | `root` | 数据库用户名 |
| `password` | string | - | 数据库密码 |
| `database` | string | **必填** | 数据库名称 |
| `pool_size` | int | `5` | 连接池大小，建议 5-10 |
| `pool_recycle` | int | `3600` | 连接回收时间(秒)，防止连接超时 |

**示例**：
```json
"database": {
  "driver": "mysql+pymysql",
  "host": "localhost",
  "port": 3306,
  "username": "root",
  "password": "your_password",
  "database": "wolin_db",
  "pool_size": 5,
  "pool_recycle": 3600
}
```

生成的完整 URL：`mysql+pymysql://root:your_password@localhost:3306/wolin_db`

---

### 2. jwt - JWT认证配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `secret_key` | string | **必填** | JWT 签名密钥，至少 16 字符 |
| `algorithm` | string | `HS256` | 加密算法，推荐 HS256 |
| `access_token_expire_minutes` | int | `1440` | Token 过期时间(分钟)，默认 1 天 |

**生成随机密钥**：
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

**示例**：
```json
"jwt": {
  "secret_key": "your-secret-key-at-least-16-characters",
  "algorithm": "HS256",
  "access_token_expire_minutes": 1440
}
```

---

### 3. api_keys - 第三方API密钥

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `dashscope` | string | `""` | 阿里云 DashScope API 密钥（用于向量嵌入） |
| `deepseek` | string | `""` | DeepSeek API 密钥（用于AI问答） |

**获取方式**：
- DashScope: [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/)
- DeepSeek: [DeepSeek 开放平台](https://platform.deepseek.com/)

**示例**：
```json
"api_keys": {
  "dashscope": "sk-xxxxxxxxxxxxxxxxxxxxxxxx",
  "deepseek": "sk-xxxxxxxxxxxxxxxxxxxxxxxx"
}
```

> **注意**: 不配置 API 密钥时，系统其他功能（学生管理、成绩管理等）仍可正常使用，仅 AI 问答和知识库功能不可用。

---

### 4. app - 应用配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `host` | string | `0.0.0.0` | 服务监听地址，`0.0.0.0` 表示接受所有网络访问 |
| `port` | int | `8080` | 服务端口 |
| `debug` | bool | `false` | 调试模式，生产环境务必设为 `false` |
| `title` | string | `学生管理系统` | 应用标题，显示在 Swagger 文档页 |
| `version` | string | `1.0.0` | 版本号 |

**示例**：
```json
"app": {
  "host": "0.0.0.0",
  "port": 8080,
  "debug": false,
  "title": "沃林学生管理系统",
  "version": "1.0.0"
}
```

## 环境切换

### 开发环境 vs 生产环境

可以通过环境变量指定不同的配置文件：

```bash
# Linux/Mac
export CONFIG_PATH=/path/to/prod.config.json

# Windows CMD
set CONFIG_PATH=C:\path\to\prod.config.json

# Windows PowerShell
$env:CONFIG_PATH="C:\path\to\prod.config.json"
```

### 多环境配置示例

```
config/
  config.dev.json      # 开发环境
  config.test.json     # 测试环境
  config.prod.json     # 生产环境
```

## 代码中使用配置

```python
from core.settings import get_settings

settings = get_settings()

# 数据库
db_url = settings.database.url
db_host = settings.database.host

# JWT
jwt_secret = settings.jwt.secret_key
jwt_expire = settings.jwt.access_token_expire_minutes

# API密钥
dashscope_key = settings.api_keys.dashscope
deepseek_key = settings.api_keys.deepseek

# 应用
app_host = settings.app.host
app_port = settings.app.port
```

## 常见问题

**Q: 缺少必填项会怎样？**
A: 启动时会报错，Pydantic 会提示缺少的字段。

**Q: 可以只配置部分选项吗？**
A: 可以，未配置的项使用默认值（`database` 必须指定 `database` 字段）。

**Q: 生产环境要注意什么？**
- `app.debug` 设为 `false`
- `jwt.secret_key` 使用强随机密钥
- `database.password` 不要使用弱密码
