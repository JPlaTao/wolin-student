# 用户身份管理系统使用指南

## 概述

本系统实现了基于角色的访问控制（RBAC），支持四种用户角色：普通用户（user）、学生（student）、教师（teacher）和管理员（admin）。

## 角色定义

### 1. 管理员（admin）
- **权限**：最高权限，可以管理所有用户
- **功能**：
  - 查看所有用户信息
  - 创建新用户（可指定角色）
  - 编辑用户信息（修改角色、启用/禁用）
  - 删除用户（不能删除自己）
  - 访问所有系统功能

### 2. 教师（teacher）
- **权限**：中等权限
- **功能**：
  - 查看用户列表（只读）
  - 管理学生信息
  - 管理成绩
  - 管理就业信息

### 3. 学生（student）
- **权限**：基础权限
- **功能**：
  - 查看用户列表（只读）
  - 查看自己的信息
  - 查看成绩
  - 查看就业信息

### 4. 普通用户（user）
- **权限**：最低权限
- **功能**：
  - 查看用户列表（只读）
  - 查看仪表板
  - 使用智能问答

## 使用流程

### 1. 首次使用 - 创建管理员账号

系统已预设一个管理员账号用于首次登录：

```
用户名: admin
密码: admin123
角色: admin
```

**重要提示**：
- 首次登录后请立即修改密码
- 管理员账号密码建议使用强密码

### 2. 注册新用户

#### 方式一：前端注册界面

1. 打开登录界面，点击"立即注册"
2. 填写用户名和密码
3. 从下拉菜单选择角色（普通用户/学生/教师/管理员）
4. 点击"注册"按钮

#### 方式二：管理员创建

1. 使用管理员账号登录
2. 进入"数据管理" → "用户管理"
3. 点击"新增用户"按钮
4. 填写用户信息：
   - 用户名（必填）
   - 密码（选填，默认为 123456）
   - 角色（必选）
   - 状态（默认启用）
5. 点击"保存"

### 3. 用户登录

1. 打开登录界面
2. 输入用户名和密码
3. 点击"登录"按钮
4. 登录成功后，侧边栏显示当前用户信息

### 4. 管理员管理用户

#### 查看用户列表
- 进入"数据管理" → "用户管理"
- 系统会自动加载所有用户列表

#### 编辑用户
1. 点击用户行的"编辑"按钮
2. 修改角色或状态
3. 点击"保存"

#### 删除用户
1. 点击用户行的"删除"按钮
2. 确认删除操作
3. **注意**：不能删除自己的账号

## 权限控制

### 前端权限控制

- **管理员**：可以查看、编辑、删除用户
- **非管理员**：只能查看用户列表，操作按钮隐藏

### 后端权限控制

```python
# 获取用户列表（所有登录用户）
GET /auth/users

# 获取单个用户信息（所有登录用户）
GET /auth/users/{user_id}

# 更新用户信息（仅管理员）
PUT /auth/users/{user_id}
Requires: current_user.role == "admin"

# 删除用户（仅管理员）
DELETE /auth/users/{user_id}
Requires: current_user.role == "admin"
```

## 用户身份管理逻辑

### 注册流程

```
用户填写信息 → 前端验证 → 发送注册请求
                        ↓
              后端验证用户名是否已存在
                        ↓
            创建用户，角色使用前端传入的值
                        ↓
                    注册成功
```

### 登录流程

```
用户输入用户名密码 → 前端验证 → 发送登录请求
                            ↓
                后端验证用户名密码
                            ↓
                生成 JWT Token（包含用户名）
                            ↓
                返回 Token 和用户信息
                            ↓
            前端保存 Token 到 localStorage
                            ↓
            设置 isAdmin = (role === 'admin')
                            ↓
                    登录成功
```

### 权限检查流程

```
用户操作 → 检查是否登录 → 检查角色 → 允许/拒绝操作
                ↓              ↓
          Token 有效    role === 'admin'
```

## 常见问题

### Q: 如何重置用户密码？

**A**: 当前版本不支持密码重置功能。管理员可以删除用户并重新创建。

### Q: 如何创建多个管理员？

**A**:
1. 使用现有管理员账号登录
2. 进入"用户管理"
3. 点击"新增用户"
4. 选择角色为"管理员"
5. 保存即可

### Q: 非管理员可以提升自己的权限吗？

**A**: 不可以。只有管理员才能修改用户角色。

### Q: 忘记密码怎么办？

**A**: 使用管理员账号登录，删除该用户并重新创建。

### Q: 为什么不能删除自己的账号？

**A**: 为了防止管理员误操作导致无法管理系统，系统禁止删除当前登录的用户账号。

## 安全建议

1. **密码安全**
   - 使用强密码（至少 8 位，包含大小写字母、数字和特殊字符）
   - 定期更换密码
   - 不要与他人共享账号

2. **权限管理**
   - 根据实际需求分配角色
   - 定期检查用户权限
   - 禁用不活跃的用户账号

3. **系统维护**
   - 首次登录后修改默认管理员密码
   - 定期备份用户数据
   - 监控异常登录行为

## 技术实现

### 前端
- 使用 Vue 3 状态管理：`isLoggedIn`、`currentUser`、`isAdmin`
- 使用 Element Plus 组件库提供 UI
- 使用 Axios 发送 HTTP 请求
- Token 存储在 `localStorage`

### 后端
- 使用 FastAPI 提供 REST API
- 使用 JWT（JSON Web Token）进行认证
- 使用 SQLAlchemy 操作数据库
- 使用 Pydantic 进行数据验证

### 数据库
- 用户表（users）包含以下字段：
  - `id`: 主键
  - `username`: 用户名（唯一）
  - `hashed_password`: 加密密码
  - `role`: 角色（user/student/teacher/admin）
  - `is_active`: 是否启用

## API 端点

### 认证相关
- `POST /auth/register` - 注册新用户
- `POST /auth/login` - 用户登录
- `GET /auth/me` - 获取当前用户信息
- `POST /auth/logout` - 用户退出（前端清除 Token）

### 用户管理
- `GET /auth/users` - 获取用户列表
- `GET /auth/users/{user_id}` - 获取单个用户信息
- `PUT /auth/users/{user_id}` - 更新用户信息（仅管理员）
- `DELETE /auth/users/{user_id}` - 删除用户（仅管理员）
