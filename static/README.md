# 前端代码结构说明

## 目录结构

```
static/
├── index.html          # 主 HTML 文件（页面结构）
├── css/
│   └── style.css       # 全局样式文件
└── js/
    └── app.js          # Vue 3 应用主文件
```

## 文件说明

### `index.html`
页面的 HTML 结构文件，包含：
- 登录/注册模态框
- 主界面布局（侧边栏 + 主内容区）
- 数据看板、智能问答、高级统计、数据管理等各个模块的模板

**职责**：定义页面结构和组件模板

### `css/style.css`
全局样式文件，包含：
- 全局样式设置
- 滚动条样式
- 卡片样式
- 聊天气泡样式
- 模态框样式
- 图表样式
- 表格样式
- Element Plus 主题覆盖
- 自定义工具类（角色标签、状态指示器等）
- 响应式调整
- 动画效果

**职责**：管理所有页面的视觉样式

### `js/app.js`
Vue 3 应用主文件，包含：
- 状态管理
- 认证逻辑
- 用户管理
- 数据看板
- 智能问答
- 数据管理（学生、班级、教师、用户、成绩、就业）
- 图表渲染
- 生命周期钩子

**职责**：处理所有业务逻辑和交互

## 新增功能

### 1. 用户管理模块

**后端 API**（`api/auth_api.py`）：
- `GET /auth/users` - 获取用户列表
- `GET /auth/users/{user_id}` - 获取单个用户信息
- `PUT /auth/users/{user_id}` - 更新用户信息（仅管理员）
- `DELETE /auth/users/{user_id}` - 删除用户（仅管理员）

**前端界面**：
- 用户列表展示
- 用户信息编辑（角色、状态）
- 用户创建
- 用户删除（带确认）
- 角色标签显示（管理员、教师、学生、普通用户）
- 状态指示器（启用/禁用）

### 2. 用户信息展示

在侧边栏底部显示当前登录用户信息：
- 用户头像（首字母）
- 用户名
- 角色（带颜色标签）
- 退出登录按钮

## 角色权限

### 角色类型
- `admin` - 管理员（红色标签）
- `teacher` - 教师（橙色标签）
- `student` - 学生（绿色标签）
- `user` - 普通用户（灰色标签）

### 权限说明
- 所有登录用户可以查看基本信息
- 管理员可以管理用户（创建、编辑、删除、修改角色和状态）
- 普通用户不能删除自己的账号

## 设计规范

### 1. 代码组织
- HTML 负责结构
- CSS 负责样式
- JS 负责逻辑
- 职责清晰，易于维护

### 2. 命名规范
- 文件名：小写 + 下划线（如 `style.css`）
- CSS 类名：短横线分隔（如 `user-info-card`）
- JS 变量：驼峰命名（如 `isLoggedIn`）
- JS 函数：驼峰命名（如 `loadUsers`）

### 3. 注释规范
- 重要的代码块使用区块注释
- 函数使用 JSDoc 风格的注释
- 复杂逻辑使用行内注释

### 4. 样式规范
- 使用 CSS 变量管理主题色
- 响应式设计使用媒体查询
- 动画使用 CSS transitions
- 避免内联样式（特殊情况除外）

## 技术栈

- **Vue 3** - 渐进式 JavaScript 框架
- **Element Plus** - Vue 3 组件库
- **Axios** - HTTP 客户端
- **ECharts** - 数据可视化库
- **Tailwind CSS** - 实用工具优先的 CSS 框架
- **Font Awesome** - 图标库

## 开发建议

### 1. 本地开发
```bash
# 启动后端服务
python main.py

# 访问前端
http://127.0.0.1:8080/static/index.html
```

### 2. 修改样式
1. 编辑 `css/style.css`
2. 刷新浏览器查看效果

### 3. 修改逻辑
1. 编辑 `js/app.js`
2. 刷新浏览器查看效果
3. 使用浏览器开发者工具调试

### 4. 修改结构
1. 编辑 `index.html`
2. 确保保留必要的 CDN 链接
3. 确保引入 CSS 和 JS 文件

## 注意事项

1. **认证令牌**：使用 localStorage 存储访问令牌
2. **会话管理**：智能问答使用 session_id 实现多轮对话
3. **错误处理**：所有 API 调用都包含错误处理
4. **加载状态**：异步操作显示加载指示器
5. **用户确认**：删除操作需要用户确认

## 性能优化

1. **懒加载**：图表按需初始化
2. **请求合并**：使用 Promise.all 并发请求
3. **防抖处理**：搜索框输入防抖
4. **图表销毁**：切换标签时销毁旧图表实例

## 后续优化方向

1. **代码分割**：将 JS 按模块拆分
2. **样式优化**：提取公共样式到变量
3. **组件化**：将重复代码封装为组件
4. **状态管理**：引入 Pinia 进行状态管理
5. **路由管理**：引入 Vue Router 实现单页应用
6. **类型检查**：引入 TypeScript
7. **构建工具**：使用 Vite 进行项目构建
8. **单元测试**：添加组件和函数测试

## 常见问题

### Q: 如何修改主题色？
A: 在 `css/style.css` 中修改 CSS 变量或直接修改颜色值。

### Q: 如何添加新功能？
A: 
1. 后端：在对应的 API 文件中添加端点
2. 前端：在 `js/app.js` 中添加对应的状态和函数
3. 界面：在 `index.html` 中添加对应的模板

### Q: 如何调试？
A: 使用浏览器开发者工具（F12）查看 Console 和 Network 面板。
